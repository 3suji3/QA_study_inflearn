# 작성 규칙

강의 노트는 나중에 다시 읽기 쉬운 공개 문서 형태로 정리합니다.

## 기본 구조

1. 핵심 요약
2. 주요 개념
3. 실무 체크포인트
4. 예시 또는 템플릿
5. 복습 질문

## 정리 원칙

- 강의 문장을 길게 그대로 옮기지 않습니다.
- 개념보다 먼저 “실무에서 왜 중요한지”를 드러냅니다.
- 긴 문단보다 표, 체크리스트, 짧은 문장으로 정리합니다.
- 기능별로 성공 조건, 실패 조건, 확인 포인트를 나눕니다.
- `class` 폴더 안에 강의별 노트를 차곡차곡 추가합니다.

## 추천 파일명

가능하면 영어 소문자와 하이픈을 사용합니다.

```text
class/qa-daily-flow.md
class/test-case-basics.md
class/bug-report-writing.md
```
