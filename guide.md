# 작성 규칙

각 강의 노트는 아래 흐름을 기준으로 정리합니다.

## 기본 구조

1. 한 문장 정의
2. 핵심 개념
3. 실무 예시
4. 헷갈리는 점
5. 다음에 확인할 질문

## 정리 원칙

- 강의 문장을 길게 그대로 옮기지 않습니다.
- 중요한 표현이 필요하면 짧은 인용만 남기고, 어느 강의에서 나온 내용인지 적습니다.
- 나중에 찾기 쉽도록 제목과 파일명은 내용을 바로 떠올릴 수 있게 씁니다.
- `class` 폴더 안에 강의별 노트를 차곡차곡 추가합니다.

## 추천 파일명

가능하면 영어 소문자와 하이픈을 사용합니다.

```text
class/qa-daily-flow.md
class/test-case-basics.md
class/bug-report-writing.md
```
