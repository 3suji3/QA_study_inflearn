# QA 영어 기초 로드맵

::: tip 핵심 요약
QA 영어는 문법보다 실무에서 반복되는 단어와 문장 패턴을 먼저 익히는 것이 중요하다. 테스트 문서, 버그 리포트, 협업 코멘트에서 자주 쓰는 표현을 상황별로 공부하면 된다.
:::

## 이렇게 공부하기

영어 초보자라면 모든 단어를 한 번에 외우려고 하기보다, QA 업무 흐름에 맞춰 나누어 보는 편이 좋다.

| 순서 | 문서 | 목표 |
| --- | --- | --- |
| 1 | [QA 기본 용어](/appendix/qa-english-terms) | 테스트 종류, 우선순위, 위험도 같은 핵심 단어 이해 |
| 2 | [테스트 케이스 영어](/appendix/qa-english-test-cases) | 테스트 케이스 문서를 읽고 작성하는 표현 익히기 |
| 3 | [버그 리포트 영어](/appendix/qa-english-bug-reports) | 버그 제목, 재현 절차, 기대/실제 결과 표현 익히기 |
| 4 | [협업 영어 표현](/appendix/qa-english-collaboration) | Jira, Slack, PR 코멘트에서 쓰는 상태와 요청 표현 익히기 |

## 먼저 외울 핵심 10개

| 영어 | 한국어 의미 | 실무에서 보는 곳 |
| --- | --- | --- |
| Test Case | 테스트 케이스 | 테스트 문서 |
| Test Scope | 테스트 범위 | 테스트 계획 |
| Preconditions | 사전 조건 | 테스트 케이스 |
| Test Steps | 테스트 절차 | 테스트 케이스 |
| Expected Result | 기대 결과 | 테스트 케이스, 버그 리포트 |
| Actual Result | 실제 결과 | 테스트 결과, 버그 리포트 |
| Reproduce | 재현하다 | 버그 리포트 |
| Severity | 심각도 | 이슈 관리 |
| Priority | 우선순위 | 이슈 관리 |
| Regression Test | 회귀 테스트 | 수정 후 재검증 |

## 영어 문제를 만났을 때 읽는 순서

1. `NOT`, `except`, `fail`처럼 반대 의미가 있는 단어를 먼저 찾는다.
2. `test scope`, `expected result`, `severity`처럼 QA 키워드를 찾는다.
3. 주어와 동사를 완벽히 해석하려고 하기보다 “무엇을 묻는지”부터 잡는다.
4. 보기를 전부 번역하지 말고, 아는 QA 개념과 연결되는 선택지를 먼저 표시한다.
5. 모르는 표현은 문장 전체보다 업무 상황으로 추측한다.

::: info 실무 포인트
QA 영어는 “영어 공부”라기보다 “업무 용어 공부”에 가깝다. 같은 표현이 테스트 케이스, Jira, Slack에서 반복해서 나오기 때문에 자주 보는 표현부터 익히면 된다.
:::

## 자주 헷갈리는 표현

| 표현 | 헷갈리는 이유 | 구분 |
| --- | --- | --- |
| Severity / Priority | 둘 다 중요해 보임 | 심각도는 영향, 우선순위는 처리 순서 |
| Expected / Actual | 둘 다 결과라는 뜻 | Expected는 기대 결과, Actual은 실제 결과 |
| Re-test / Regression Test | 둘 다 다시 테스트함 | Re-test는 수정 이슈 확인, Regression은 주변 영향 확인 |
| Bug / Issue / Defect | 모두 문제처럼 보임 | 팀마다 다르게 쓰지만 Issue가 가장 넓은 표현 |
