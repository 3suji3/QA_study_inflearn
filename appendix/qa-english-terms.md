# QA 기본 용어

::: tip 핵심 요약
QA 실무 영어의 시작은 테스트 종류, 우선순위, 위험도, 결과 판정에 쓰이는 단어를 익히는 것이다.
:::

## 테스트 계획과 범위

| 영어 | 한국어 의미 | 쉬운 설명 |
| --- | --- | --- |
| Test Plan | 테스트 계획 | 무엇을, 언제, 어떻게 테스트할지 정리한 계획 |
| Test Scope | 테스트 범위 | 이번에 어디까지 테스트할지 |
| Out of Scope | 테스트 제외 범위 | 이번 테스트에서 제외하는 부분 |
| Requirement | 요구사항 | 기능이 만족해야 하는 조건 |
| Acceptance Criteria | 인수 기준 | 이 조건을 만족하면 완료로 볼 수 있는 기준 |
| Test Environment | 테스트 환경 | OS, 브라우저, 서버, 앱 버전 등 |
| Test Data | 테스트 데이터 | 테스트에 사용하는 계정, 값, 파일 등 |

### 예문

| 영어 문장 | 뜻 |
| --- | --- |
| This feature is in scope for this release. | 이 기능은 이번 릴리스 테스트 범위에 포함됩니다. |
| Payment refund is out of scope for this cycle. | 환불 기능은 이번 테스트 사이클 범위에서 제외됩니다. |
| Please confirm the acceptance criteria. | 완료 기준을 확인해주세요. |

## 우선순위와 위험도

| 영어 | 한국어 의미 | 쉬운 설명 |
| --- | --- | --- |
| Priority | 우선순위 | 얼마나 빨리 처리해야 하는지 |
| Severity | 심각도 | 문제가 서비스에 얼마나 큰 영향을 주는지 |
| Risk | 위험 | 문제가 생길 가능성 또는 영향 |
| User Impact | 사용자 영향도 | 사용자가 얼마나 불편을 겪는지 |
| Critical | 치명적인, 매우 중요한 | 서비스 핵심 흐름이 막히는 수준 |
| Blocker | 진행 차단 이슈 | 다음 단계로 진행할 수 없게 만드는 문제 |
| Workaround | 우회 방법 | 임시로 문제를 피하는 방법 |

### Severity와 Priority 차이

| 구분 | 의미 | 예시 |
| --- | --- | --- |
| Severity | 문제의 영향 크기 | 결제 완료 후 주문이 생성되지 않음 |
| Priority | 수정 순서 | 오늘 배포 전 반드시 수정해야 함 |

심각도가 높아도 발생 조건이 매우 제한적이면 우선순위가 조정될 수 있다. 반대로 심각도는 낮아도 많은 사용자가 매일 겪는 문제라면 우선순위가 높아질 수 있다.

## 테스트 종류

| 영어 | 한국어 의미 | 언제 쓰는지 |
| --- | --- | --- |
| Smoke Test | 스모크 테스트 | 빌드가 기본적으로 테스트 가능한 상태인지 빠르게 확인 |
| Sanity Test | 새니티 테스트 | 특정 수정이나 변경이 말이 되는지 좁게 확인 |
| Regression Test | 회귀 테스트 | 수정 후 기존 기능이 깨지지 않았는지 확인 |
| Functional Test | 기능 테스트 | 기능이 요구사항대로 동작하는지 확인 |
| Non-functional Test | 비기능 테스트 | 성능, 보안, 접근성 같은 품질 속성 확인 |
| Exploratory Testing | 탐색적 테스트 | 정해진 케이스 밖에서 의심하며 탐색 |
| Ad-hoc Testing | 임의 테스트 | 준비된 절차 없이 즉석에서 확인 |
| UAT | 사용자 인수 테스트 | 실제 사용자나 현업 관점에서 완료 여부 확인 |

## 입력값 테스트 기법

| 영어 | 한국어 의미 | 쉬운 설명 |
| --- | --- | --- |
| Equivalence Partitioning | 동등 분할 | 비슷한 값들을 그룹으로 나누어 대표값을 테스트 |
| Boundary Value Analysis | 경계값 분석 | 7, 8, 16, 17처럼 경계 주변 값을 테스트 |
| Valid Input | 유효한 입력 | 허용되는 값 |
| Invalid Input | 유효하지 않은 입력 | 거부되어야 하는 값 |
| Required Field | 필수 입력 항목 | 반드시 입력해야 하는 값 |

## 결과 판정

| 영어 | 한국어 의미 | 설명 |
| --- | --- | --- |
| Pass | 통과 | 기대 결과와 실제 결과가 일치 |
| Fail | 실패 | 실제 결과가 기대 결과와 다름 |
| Not Tested | 미실행 | 아직 테스트하지 않음 |
| Blocked | 차단됨 | 다른 문제 때문에 테스트를 진행할 수 없음 |
| N/A | 해당 없음 | 이 케이스에는 적용되지 않음 |
