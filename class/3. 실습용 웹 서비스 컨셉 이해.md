# 실습용 웹 서비스 컨셉 이해

::: tip 핵심 요약
회원가입, 로그인, 마이페이지는 사용자 흐름이 명확하고 예외 상황이 많아서 QA 테스트 설계를 연습하기 좋은 기능이다.
:::

## 실습 대상 기능

실습용 웹 서비스는 아래 기능을 중심으로 이해한다.

| 기능 | 테스트하기 좋은 이유 |
| --- | --- |
| 회원가입 | 대부분의 서비스에 공통으로 있고 입력값 검증이 많음 |
| 로그인 | 성공과 실패 결과가 명확해서 설명하기 좋음 |
| 마이페이지 | 사용자 정보 조회, 수정, 비밀번호 변경 등 테스트 포인트가 많음 |

## 테스트 케이스를 바라보는 관점

테스트 케이스는 단순히 화면 흐름을 따라가는 문서가 아니다.

기능이 성공적으로 작동하는 조건은 무엇인지, 사용자가 실수했을 때 서비스가 어떻게 대응해야 하는지를 미리 생각하고 정리하는 과정이다.

## 회원가입 테스트 관점

회원가입은 정보 입력부터 가입 완료까지 여러 단계가 이어진다.

| 단계 | 확인할 것 | 예외 케이스 |
| --- | --- | --- |
| 정보 입력 | 필수값이 모두 입력되었는지 | 필수값 누락 |
| 유효성 검사 | 이메일과 비밀번호 규칙을 만족하는지 | 이메일 형식 오류, 비밀번호 규칙 미준수 |
| 중복 확인 | 이미 가입된 이메일인지 | 중복 이메일 |
| 약관 동의 | 필수 약관에 동의했는지 | 필수 약관 미동의 |
| 가입 완료 | 성공 메시지와 이동 경로가 정상인지 | 완료 처리 실패 |

회원가입 테스트에서는 각 단계마다 `성공 조건`과 `실패 조건`을 나누어 생각해야 한다.

## 로그인 테스트 관점

로그인은 정상 흐름과 실패 흐름이 분명하다.

| 흐름 | 확인할 것 |
| --- | --- |
| 로그인 성공 | 마이페이지 또는 지정된 화면으로 이동하는지 |
| 로그인 실패 | 올바른 에러 메시지가 표시되는지 |
| 비밀번호 찾기 | 로그인 실패 후 회복 흐름이 제공되는지 |
| 세션 관리 | 로그인 상태 유지와 만료 기준이 의도대로 동작하는지 |

## 마이페이지 테스트 관점

마이페이지는 사용자가 자신의 핵심 정보를 직접 다루는 영역이다.

| 기능 | 확인할 것 |
| --- | --- |
| 사용자 정보 조회 | 저장된 데이터가 정확하게 표시되는지 |
| 정보 수정 | 수정 후 저장 처리가 정상적으로 되는지 |
| 유효성 검사 | 잘못된 값 입력 시 적절히 막히는지 |
| 비밀번호 변경 | 기존 비밀번호 확인, 새 비밀번호 규칙, 변경 완료 처리가 정상인지 |

::: info 실무 포인트
마이페이지처럼 개인정보와 연결된 기능은 “정상 동작”뿐 아니라 “실패했을 때 사용자의 정보가 안전하게 유지되는지”도 함께 봐야 한다.
:::

## 웹 서비스 공통 확인 포인트

회원가입, 로그인, 마이페이지처럼 계정과 연결된 기능을 볼 때는 기능별 흐름 외에도 공통으로 확인할 항목이 있다.

| 관점 | 확인할 것 |
| --- | --- |
| 권한 | 로그인하지 않은 사용자가 접근할 수 없는 화면에 들어갈 수 없는지 |
| 상태 유지 | 새로고침, 뒤로 가기, 탭 이동 후에도 상태가 의도대로 유지되는지 |
| 에러 메시지 | 사용자가 다음 행동을 알 수 있을 만큼 메시지가 명확한지 |
| 데이터 보존 | 실패 상황에서 사용자가 입력한 값이 불필요하게 사라지지 않는지 |
| 중복 처리 | 버튼 연속 클릭이나 중복 요청으로 데이터가 두 번 생성되지 않는지 |

이런 항목은 특정 기능 하나만의 문제가 아니라 웹 서비스 전반에서 반복해서 확인해야 하는 기본 테스트 관점이다.

## 면접 답변 예시

회원가입 기능을 기준으로 사용자 영향이 큰 흐름부터 테스트했고, 정상 케이스와 예외 케이스로 나누어 테스트 케이스를 설계했습니다.

## 복습 질문

- 회원가입에서 사용자 영향이 가장 큰 실패 케이스는 무엇일까?
- 로그인 실패 메시지는 보안과 사용자 편의 사이에서 어디까지 자세히 보여줘야 할까?
- 마이페이지 정보 수정 후 저장 실패가 발생하면 입력한 값은 어떻게 유지되어야 할까?
- 로그인하지 않은 사용자가 마이페이지 URL로 직접 접근하면 어떻게 처리되어야 할까?
