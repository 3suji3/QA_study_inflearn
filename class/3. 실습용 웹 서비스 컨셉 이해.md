# 실습용 웹 서비스 컨셉 이해

## 한 문장 정의

실습용 웹 서비스는 회원가입, 로그인, 마이페이지처럼 대부분의 서비스에 공통으로 존재하는 기능을 기준으로 QA 테스트 흐름을 연습하기 위한 대상이다.

## 핵심 개념

실습에서 다루기 좋은 기능은 사용자 흐름이 분명하고, 정상 케이스와 예외 케이스를 나누기 쉬운 기능이다.

- 회원가입: 대부분의 회사 서비스에 공통으로 존재한다.
- 로그인: 면접에서 테스트 설계 과정을 설명하기 좋다.
- 마이페이지: 사용자 정보 조회, 수정, 비밀번호 변경 등 테스트 포인트가 많다.

테스트 케이스는 단순히 기능의 화면 흐름을 따라가는 문서가 아니다. 서비스의 다양한 흐름과 예외 상황을 미리 생각하고, 어떤 조건에서 성공 또는 실패로 판단할지 정리하는 과정이다.

## 예시

### 회원가입

회원가입은 정보 입력부터 가입 완료까지 여러 단계가 이어지는 기능이다.

- 정보 입력: 필수값이 모두 입력되었는지 확인한다.
- 유효성 검사: 이메일 형식, 비밀번호 규칙, 이미 가입된 이메일 여부를 확인한다.
- 약관 동의: 필수 약관에 동의하지 않았을 때 가입이 막히는지 확인한다.
- 가입 완료: 성공 메시지 또는 완료 화면이 정상적으로 표시되는지 확인한다.

회원가입 테스트에서는 각 단계마다 `성공 조건은 무엇인가?`, `실패하는 경우는 무엇인가?`를 계속 나누어 생각해야 한다.

### 로그인

로그인은 성공과 실패 결과가 명확해서 테스트 설계를 설명하기 좋다.

- 로그인 성공: 마이페이지 또는 지정된 화면으로 이동하는지 확인한다.
- 로그인 실패: 올바른 에러 메시지가 표시되는지 확인한다.
- 비밀번호 찾기: 사용자가 로그인하지 못했을 때 회복 흐름이 제공되는지 확인한다.
- 세션 관리: 로그인 상태가 유지되거나 만료되는 기준이 의도대로 동작하는지 확인한다.

이 과정은 기능이 성공적으로 작동하는 조건과, 사용자가 실수했을 때 서비스가 어떻게 대응해야 하는지를 미리 생각하는 연습이 된다.

### 마이페이지

마이페이지는 사용자가 자신의 핵심 정보를 직접 다루는 영역이다.

- 사용자 정보 조회: 저장된 데이터가 정확하게 표시되는지 확인한다.
- 정보 수정: 수정 후 저장 처리가 정상적으로 되는지 확인한다.
- 유효성 검사: 잘못된 값으로 수정할 때 적절히 막히는지 확인한다.
- 비밀번호 변경: 기존 비밀번호 확인, 새 비밀번호 규칙, 변경 완료 처리를 확인한다.

마이페이지 테스트에서는 `이 기능은 언제 성공인가?`, `실패는 어떤 경우인가?`, `사용자가 실수하면 어떻게 되는가?`를 세심하게 확인해야 한다.

## 헷갈리는 점

기능 흐름을 그대로 따라가는 것과 테스트 케이스를 설계하는 것은 다르다. QA는 화면 순서를 기록하는 데서 멈추지 않고, 정상 흐름과 예외 흐름을 나누어 위험한 지점을 찾아야 한다.

## 질문

- 회원가입에서 가장 사용자 영향이 큰 실패 케이스는 무엇일까?
- 로그인 실패 메시지는 보안과 사용자 편의 사이에서 어디까지 자세히 보여줘야 할까?
- 마이페이지 정보 수정 후 저장 실패가 발생하면 사용자가 입력한 값은 어떻게 유지되어야 할까?

## 면접 답변 예시

회원가입 기능 기준으로 사용자 영향이 큰 흐름부터 테스트했고, 정상 케이스와 예외 케이스로 나누어 테스트 케이스를 설계했습니다.
