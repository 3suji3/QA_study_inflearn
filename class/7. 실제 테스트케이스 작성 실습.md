# 실제 테스트케이스 작성 실습

::: tip 핵심 요약
테스트 케이스는 테스트 목적, 사전 조건, 테스트 절차, 기대 결과를 기준으로 기능의 성공과 실패를 검증 가능한 문서로 만드는 작업이다.
:::

## 테스트 케이스의 4가지 필수 요소

테스트 케이스에는 최소한 아래 4가지가 들어가야 한다.

| 항목 | 의미 | 예시 |
| --- | --- | --- |
| 테스트 목적 | 무엇을 확인하려는지 | 회원가입 기능이 정상 동작하는지 확인 |
| 사전 조건 | 테스트 전 필요한 상태 | 회원가입 페이지 접속, 인터넷 연결 |
| 테스트 절차 | 어떤 순서로 실행할지 | 이메일 입력 -> 비밀번호 입력 -> 약관 동의 -> 가입 버튼 클릭 |
| 기대 결과 | 정상이라면 어떤 결과가 나와야 하는지 | 가입 완료 메시지 표시, 로그인 페이지 이동 |

이 4가지가 있어야 테스트 결과를 `Pass` 또는 `Fail`로 판단할 수 있다.

## 테스트 케이스에 담아야 하는 질문

회원가입 기능을 테스트 케이스로 만들 때는 아래 질문에서 시작한다.

| 질문 | 테스트로 바뀌는 방향 |
| --- | --- |
| 언제 성공인가? | 정상 흐름 테스트 |
| 실패는 어떤 경우인가? | 입력값 유효성 테스트, 예외 상황 테스트 |
| 사용자가 실수하면 어떻게 되는가? | 오류 메시지, 입력값 보존, 재시도 가능 여부 확인 |

테스트 케이스는 기능 설명을 옮겨 적는 문서가 아니다. QA가 던진 질문을 실행 가능한 절차와 기대 결과로 바꾸는 문서다.

## 회원가입 정상 흐름 테스트

정상 흐름 테스트는 사용자가 기대하는 방식대로 기능이 올바르게 동작하는지 확인한다.

| 단계 | 테스트 절차 | 기대 결과 |
| --- | --- | --- |
| 1 | 회원가입 페이지에 접속한다. | 회원가입 입력 화면이 표시된다. |
| 2 | 이메일, 비밀번호 등 필수 정보를 입력한다. | 입력값이 정상적으로 입력된다. |
| 3 | 필수 약관에 동의한다. | 약관 동의 상태가 반영된다. |
| 4 | 가입하기 버튼을 클릭한다. | 회원가입 요청이 처리된다. |
| 5 | 가입 완료 화면을 확인한다. | 성공 메시지 또는 환영 메시지가 표시된다. |

::: info 실무 포인트
정상 흐름은 가장 기본이지만 가장 중요하다. 정상 흐름이 깨져 있으면 예외 케이스를 확인하기 전에 핵심 기능 자체가 동작하지 않는 상태다.
:::

## 입력값 유효성 테스트

입력값 유효성 테스트는 다양한 입력 규칙에 따라 값이 올바르게 처리되는지 확인한다.

| 케이스 | 테스트 데이터 | 기대 결과 |
| --- | --- | --- |
| 필수 항목 누락 | 이메일 미입력 | `이메일을 입력해주세요.` 메시지 표시 |
| 이메일 형식 오류 | `test-email` | 이메일 형식 오류 메시지 표시 |
| 비밀번호 규칙 위반 | 8자 미만 비밀번호 | `비밀번호는 8자 이상이어야 합니다.` 메시지 표시 |
| 중복 이메일 | 이미 가입된 이메일 | `이미 사용 중인 이메일입니다.` 메시지 표시 |

입력값 유효성 테스트에서는 오류 메시지가 뜨는지만 보는 것이 아니라, 가입 처리가 실제로 차단되는지도 함께 확인해야 한다.

## 예외 상황 테스트

예외 상황 테스트는 예상치 못한 오류가 발생했을 때 시스템이 적절히 반응하는지 확인한다.

| 예외 상황 | 확인할 것 |
| --- | --- |
| 네트워크 오류 | 사용자에게 네트워크 문제를 안내하는지 |
| 서버 타임아웃 | 요청 실패 메시지와 재시도 가능 여부를 제공하는지 |
| 잘못된 접근 | 접근 권한이 없는 흐름을 차단하는지 |
| 시스템 오류 | 데이터가 중복 저장되거나 깨지지 않는지 |

예외 상황에서는 사용자 안내와 데이터 무결성을 함께 봐야 한다. 단순히 에러가 발생하고 끝나는 것이 아니라, 사용자가 다음 행동을 알 수 있어야 하고 시스템 상태도 안전해야 한다.

## 회원가입 오류 시나리오 비교

회원가입 오류 시나리오는 아래처럼 정리할 수 있다.

| 시나리오 | 입력 또는 상황 | 기대 결과 |
| --- | --- | --- |
| 필수 입력 누락 | 이메일 미입력 | 이메일 입력 안내 표시 |
| 비밀번호 규칙 위반 | 8자 미만 비밀번호 입력 | 비밀번호 규칙 안내 표시 |
| 중복된 정보 | 이미 사용 중인 이메일 입력 | 중복 이메일 안내 표시 |
| 가입 중 오류 | 서버 오류 또는 네트워크 오류 | 실패 안내, 중복 계정 생성 방지 |

## 테스트 케이스 작성 예시

| 항목 | 내용 |
| --- | --- |
| 테스트 ID | SIGNUP-001 |
| 분류 | 정상 흐름 |
| 테스트 목적 | 필수 정보를 모두 입력했을 때 회원가입이 성공하는지 확인 |
| 사전 조건 | 회원가입 페이지에 접속한 상태 |
| 테스트 데이터 | 신규 이메일, 유효한 비밀번호, 필수 약관 동의 |
| 테스트 절차 | 필수 정보 입력 -> 약관 동의 -> 가입하기 클릭 |
| 기대 결과 | 가입 완료 메시지가 표시되고 로그인 페이지로 이동 |
| 실제 결과 | 테스트 실행 후 기록 |
| 판정 | Pass 또는 Fail |
