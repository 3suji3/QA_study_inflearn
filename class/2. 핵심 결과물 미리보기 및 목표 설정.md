# 핵심 결과물 미리보기 및 목표 설정

::: tip 핵심 요약
QA 실무 학습의 목표는 감으로 테스트하는 것이 아니라, 테스트 케이스와 체크리스트를 기준으로 구조적으로 검증하는 것이다.
:::

## 이번 강의의 목표

이번 강의에서는 QA 실무에서 자주 쓰는 두 가지 결과물을 미리 살펴본다.

| 결과물 | 목적 |
| --- | --- |
| 테스트 케이스 세트 | 기능을 어떤 조건과 절차로 검증할지 정리 |
| QA 실무 체크리스트 | 테스트 전, 중, 후에 빠뜨리면 안 되는 항목 정리 |

## 테스트 케이스 예시

회원가입 기능을 기준으로 테스트 케이스를 작성하면 아래처럼 정리할 수 있다.

| 항목 | 내용 |
| --- | --- |
| 테스트 목적 | 회원가입 기능이 정상적으로 작동하는지 확인 |
| 사전 조건 | 인터넷 연결, 테스트 계정 정보 준비 |
| 테스트 절차 | 회원가입 페이지 접속 -> 필수 정보 입력 -> 가입하기 버튼 클릭 |
| 기대 결과 | 회원가입 성공 메시지가 표시되고 로그인 페이지로 이동 |

테스트 케이스는 “무엇을 눌러본다”에서 끝나는 문서가 아니다. 목적, 조건, 절차, 기대 결과가 함께 있어야 테스트 결과를 객관적으로 판단할 수 있다.

## QA 실무 체크리스트

테스트는 시작 전, 진행 중, 종료 후에 확인할 항목이 다르다.

| 시점 | 확인할 것 |
| --- | --- |
| 테스트 전 | 환경 확인, 요구사항 분석, 테스트 계획 수립 |
| 테스트 중 | 입력값 처리, 시나리오 실행, 결함 발견 및 기록 |
| 테스트 후 | 결과 정리, 보고서 작성, 개선점 도출 |

## 핵심 관점

QA는 감이 아니라 구조로 일한다.

- 테스트 설계 실무: 테스트 케이스 세트를 완성한다.
- 테스트 수행 실무: QA 체크리스트를 기준으로 검증한다.
- 결과 정리: 테스트 결과와 발견한 이슈를 공유 가능한 형태로 남긴다.

::: info 실무 포인트
테스트 케이스와 체크리스트는 QA가 놓친 부분을 줄이고, 다른 사람도 같은 기준으로 테스트할 수 있게 만드는 도구다.
:::

## 복습 질문

- 테스트 케이스에 테스트 목적이 필요한 이유는 무엇일까?
- 사전 조건이 빠지면 테스트 결과 해석에 어떤 문제가 생길까?
- 체크리스트는 테스트 전, 중, 후에 각각 어떻게 달라져야 할까?
