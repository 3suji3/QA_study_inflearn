실무 QA의 하루

오늘 배포될 기능을 확인

지라 같은 프로젝트 관리 도구에서 어떤 기능이 배포 예정인지 파악하고 기능 명세서나 디자인 문서를 보며 변경된 요구사항은 없는 지 꼼꼼히 살핌

 

QA는 단순히 버그를 찾는 사람이 아니라 이 기능이 문제없이 사용자에게 나가도 되는지 판단하는 역할을 하기 때문

 

이렇게 파악한 정보를 바탕으로 디지털 체크리스트를 보완하고 테스트할 기능의 범위는 어디까지인지 그리고 가장 중요하게 테스트할 부분은 어디인지 우선순위를 결정하게 됨

실제 테스트 수행

테스트 케이스

테스트 절차, 기대 결과, 실제 결과 Pass/Fail

예상했던 결과와 오류메시지가 뜨면서 기능이 제대로 동작하지 않는다면 발견된 문제점을 실제 결과의 상세히 기록하고 fail로 표시하게 됨
